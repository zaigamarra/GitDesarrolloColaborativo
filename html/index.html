Áreas de Git

Working Directory

El area de trabajo hace referencia a la carpeta donde creamos el repositorio. Muy rara vez utilizaremos la linea de comandos en esta zona, pero es donde realizaremos todas las modificaciones relacionadas con nuestro proyecto.

  • mkdir <dir>: crea un directorio con el nombre especificado.
  • cd <dir>: este comando cambia el directorio actual por el seleccionado.
  • git init: inicializa un repositorio de git, creando una carpeta oculta que lleva el nombre "".git".
  • rm <dir>: elimina un archivo, pero podemos especificar con parametros que elimine una carpeta, por ejemplo rm -rf .git, elimina el repositorio de git.

Staging Area (INDEX)

El area de control de cambios corresponde al archivo que se encuentra en la carpeta .git que se crea al iniciar un repositorio. Aqui se registran los archivos a los que vamos a hacerle seguimiento y es el punto de comparación para confirmar dichos cambios. Es necesario agregar los archivos con los que estemos trabajando a dicha area para avanzar con el historial de cambios, ya que git al utilizar el comando git add realiza una captura del código.

Repository (Local/Remote)

El repositorio corresponde a la base de datos donde se registran o se almacenan los cambios en formato BLOG (Binary Object File) y es el lugar al que podemos acceder cada vez que necesitemos revisar algún estado anterior de nuestro proyecto. Esto es posible gracias a los commits identificados mediante un número de HASH.
